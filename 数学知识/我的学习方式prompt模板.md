# 🧠 我的学习方式 & Prompt 模板

## 我的学习者画像

我是 **对比追问式学习者**。我的学习特点是：

1. **不满足表面答案** — 需要具体化、可触摸的类比才能"真懂"
2. **靠对比来理解** — 不是单独学一个概念，而是通过并列对比两个相似概念
3. **需要"恍然大悟时刻"** — 一个具体数字对比或场景反转让我"啊！原来如此"
4. **递归追问** — 每次懂了都会暴露新的不懂，直到问到能"摸到"的程度
5. **需要统一框架** — 最后必须揭示两个概念在更高层面是统一的

## Prompt 模板（直接复制使用）

```markdown
## 学习请求

我想理解 **[概念A]** 和 **[概念B]** 的区别和联系。

## 适用我的学习方式

请按以下步骤教学：

### 第一步：用生活比喻建立直觉
不要一开始讲公式和定义。先用一个生活中的具体场景或对象来比喻这个区别，让我在脑子里有个图像。

### 第二步：并排对比表
列一个 3-5 个维度的对比表格：

| 维度 | [概念A] | [概念B] |
| --- | --- | --- |
| 核心假设 | ? | ? |
| 数学形式 | ? | ? |
| 典型场景 | ? | ? |
| 对同一个错误的惩罚差异 | ? | ? |
| "世界观" | ? | ? |

### 第三步：具体数字对比 — 制造"恍然大悟"
给我一个极端场景，展示两个概念的处理差异。
比如：同样是犯某个错误，[概念A]罚了 0.9，[概念B]罚了 3.0。
用具体数字让我"啊"一下。

### 第四步：揭示底层逻辑
为什么会有这种区别？它们分别基于什么底层假设？
从更高的统一框架（如极大似然估计）来演绎推导。

### 第五步：HTML 展示
用 HTML 页面展示以上全部内容，要求：
- 左右分栏或并排卡片布局
- 双色块区分概念（如蓝/粉）
- MathJax 渲染公式
- 具体数字例子高亮
- 亮色主题，视觉友好
- 风趣幽默的语言

### 第六步：追问留白
讲完后问我一句："你现在还有哪里没想通？"
```

## 使用示例

当你想学一个新东西时，这样用：

```
模板中的 [概念A] 和 [概念B] 替换成你想理解的概念对。

常见学习对：
- 概率 vs 似然
- MSE vs 负对数似然 (NLL)
- PPO vs DPO
- BT模型 vs 逻辑回归
- 正态噪声 vs 伯努利噪声
- 极大似然估计 vs 贝叶斯估计
- 过拟合 vs 欠拟合
- 偏差 vs 方差
- 强化学习 vs 监督学习
```

## 给大模型的提示（如果它还没理解你的方式）

如果 AI 的回答还是不够好，可以补充：

```
请记住我的学习方式：
1. 我靠对比来理解，不是靠单独定义
2. 我需要具体的数字例子来制造"恍然大悟"
3. 先用比喻，再给公式
4. 最后要告诉我它们在哪一个更高的层面是统一的
5. 用 HTML 并排展示
```
