# Deep Compare Learn — 对比追问式教学

## 这个 Skill 做什么

这个 skill 封装了一套适合**对比追问式学习者**的教学方法论。它的核心假设是：

> 用户不是通过"从定义到例子"的线性学习来理解的，而是通过 **并排对比 → 追问细节 → 恍然大悟 → 发现新盲区 → 再追问** 的递归过程。

当用户提出类似"啥是噪声""MSE 和 NLL 有啥区别""概率和似然不一样吗"这类对比性问题时，使用本 skill。

## 何时使用

**检测触发词**（用户说出以下内容时激活）：
- "A 和 B 有啥区别/联系？"
- "啥是 X？"（尤其是追问了两次以上还不满足时）
- "我没明白 X 是啥意思"
- "X 是天然的还是人为构建的？"
- "X 是啥，Y 又是啥，它们不一样吗？"
- "梳理一下"、"帮我总结"、"打通"

**典型场景**：
- 概率 vs 似然
- MSE vs 负对数似然（NLL）
- PPO vs DPO
- Bradley-Terry 模型 vs 逻辑回归
- 正态分布噪声 vs 伯努利分布噪声
- 极大似然估计 vs 贝叶斯估计
- 损失函数 vs 评价指标
- 任何"看起来像但不一样"的概念对

## 教学工作流

### Step 1 — 需求确认与学习画像定位（必做）

在开始教学前，先确认用户当前的知识状态：

```
请先回答我几个问题，这样我能用最适合你的方式来讲解：

1. [概念A] 这个名字你之前听过吗？能用自己的话描述一下吗？
2. [概念B] 呢？
3. 你主要卡在哪个环节：
   a) 不知道这些概念是啥（完全新）
   b) 大概知道是啥，但不清楚它们有啥区别
   c) 知道区别，但不明白为什么会有这种区别（底层逻辑）
   d) 知道底层逻辑，但想要一个"一以贯之"的统一框架
4. 你更喜欢哪种教学方式？
   - 从头到尾讲一个生动的故事/比喻
   - 直接给并排对比表格
   - 先给公式再解释
   - 先举具体例子，再从例子中抽象
```

如果用户说"直接讲吧"，至少确认两个信息：
1. 用户目前在哪个**卡点**（上面 a/b/c/d）
2. 用户偏好的**教学入口**（比喻/对比表/公式/例子）

### Step 2 — 确定教学策略

根据 Step 1 的定位，按以下策略递进：

**如果用户是 a) 全新概念**：
- 先用一个**生活比喻**建立直觉锚点（比如"噪声 = 尺子测量时的毛刺"）
- 然后给出**简洁定义**（一句话 + 公式）
- 然后给出**并排对比表**

**如果用户是 b) 模糊对比**：
- 直接给出**并排对比表格**，从 3-5 个维度对比
- 每个维度附一个**具体的数字例子**

**如果用户是 c) 不理解底层逻辑**：
- 核心问题通常是"为什么 X 要用这个公式/损失/假设？"
- 用**"先假设→再推导"**的链条解释
- 从**极大似然估计**这个统一框架穿针引线

**如果用户是 d) 想要统一框架**：
- 直接揭示**高层统一性**（如"所有损失函数都是某种分布假设下的极大似然"）
- 用**两张地图合并**的方式展示

### Step 3 — 追问链设计（核心差异化）

这是本 skill 区别于普通教学的核心。**用户的学习方式是递归追问式的**，所以你的教学节奏应该是：

```
比喻引入 ─→ 你没懂？→ 追问 ─→ 还没透？→ 更具体的追问 ─→ 
还是隔靴搔痒？→ 用具体数字对比制造"恍然大悟时刻"
```

**追问链模板**（顺序执行）：

1. **第一层追问**："这个 X 是天然存在的还是人为构建的？"
   → 如果是人为构建，讲清楚"它为了解决什么问题被构建出来"
   → 如果是天然存在的，给出生活实例

2. **第二层追问**："那 X 和 Y 的区别到底在哪里？"
   → 给出 3×2 对比表（3 个维度 × 2 个概念）
   → 每个维度配一个具体数字例子

3. **第三层追问**："为啥 X 用这个公式，Y 用那个公式？"
   → 揭示底层假设不同
   → 从统一的母框架（如极大似然）演绎

4. **制造"恍然大悟时刻"**：
   → 给一个**极端场景**，X 和 Y 的输出差异巨大
   → 用具体数字展示："你看，同样是这个错误，X 只罚了 0.9，Y 罚了 3.0"
   → 说"你现在懂了吧？"（用户如果真的懂了，这里会发出"啊！"的声音）

### Step 4 — HTML 输出要求

所有教学内容**必须用 HTML 页面呈现**，发挥 HTML 的视觉展示能力：

**布局要求**：
- 左右分栏或并排卡片布局
- 每个概念用独立的色块标识（如 MSE 用蓝色系，NLL 用粉色系）
- 要有对比表格区
- 要有具体数字例子区

**内容要求**：
- 数学公式用 MathJax 渲染（从 CDN 加载 `tex-svg.js`）
- 代码块用等宽字体
- 关键比喻用引用块或高亮块展示
- 语言风趣幽默，用生活场景做比喻

**样式要求**：
- 亮色主题，柔和背景（暖白、米白）
- 不刺眼的阴影和圆角
- 响应式布局（移动端可读）

**用户偏好记忆**：
- 用户喜欢"具体数字对比"的恍然大悟时刻
- 用户喜欢"并排对比"的展示方式
- 用户喜欢"先比喻再公式"的节奏
- 用户喜欢"追问到能摸到的程度"才罢休

### Step 5 — 收尾：统一框架 + 开放性追问

教学结束后，做两件事：

1. **揭示统一框架**：把两个概念放在同一个更高的框架下
   - 例如："MSE 和 NLL 都是极大似然估计在不同分布假设下的特例"
   - 这能帮助用户完成"打通"的动作

2. **留一个开放性追问**：
   - "你现在还有哪里没想通？"
   - "有没有什么新的疑问冒出来了？"
   - 用户的学习是递归的，每一次"懂了"都会暴露新的 "不懂"

## 示例教学结构（以 MSE vs NLL 为例）

```
第一段：比喻引入
  MSE → 尺子量桌子，误差是正态噪声
  NLL → 抛硬币猜公平度，结果是伯努利事件

第二段：并排对比表（3-5个维度）
  | 维度 | MSE | NLL |
  | 输出空间 | 实数 | 概率(0,1) |
  | 直觉 | 几何距离 | 概率不匹配惩罚 |
  | 底层假设 | 正态噪声 | 伯努利/BT分布 |
  | 惩罚曲线 | 二次平缓 | 对数爆炸 |

第三段：具体数字对比（制造恍然大悟）
  同一个错误：0.95→0.05
  MSE 涨 0.9，NLL 涨 2.95
  "你看，NLL 对严重错误罚的狠多了"

第四段：底层逻辑揭示
  都是极大似然估计，只是假设不同

第五段：HTML 展示（并排卡片+表格+公式+数字例子）
```
