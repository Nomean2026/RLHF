# Bradley-Terry（布拉德利 - 特里）偏好模型简要解析

## 1. 核心用途

成对比较建模：给定任意两组对象两两对决 / 两两打分偏好数据，估计每个对象的**优势参数（能力 / 吸引力得分）**，预测二者对比时一方胜出的概率。典型场景：选手比赛胜负、商品偏好排序、评委成对打分、广告效果对比。

## 2. 基础公式

设物品  的优势参数为 ，物品  与  对比， 优于  的概率：

- 越大，对象越容易胜出；
- 若 ，胜负概率各 50%。

## 3. 模型假设

1. **传递性**：若 i 优于 j、j 优于 k，则 i 大概率优于 k；
2. **独立成对比较**：每一组两两对比互不干扰；
3. 仅记录二元结果：胜 / 负，无平局（可拓展加入平局参数）。

## 4. 参数估计思路

观测数据： = i 战胜 j 的总次数， = j 战胜 i 的总次数。对数似然最大化求解 ，常用迭代算法（MM 算法）收敛得到各对象得分；参数仅相对大小有意义，通常约束  或固定某一  消除尺度不定性。

## 5. 优缺点

### 优点

- 仅需两两对比数据，不用完整排序，数据收集成本低；
- 输出可解释的连续偏好分，直接用于对象排名；
- 拓展灵活：可加协变量、分层 BT、带平局、多评判者版本。

### 局限

- 严格传递假设，现实存在循环偏好（i>j,j>k,k>i）时拟合偏差大；
- 只能处理二元胜负，无法区分优势差距大小；
- 样本过少时参数估计不稳定。

## 6. 简单拓展

1. **带平局 BT**：新增参数 ，引入双方持平概率；
2. **层次 BT**：加入评委 / 组别随机效应，处理多源打分；

# 一、Bradley-Terry（BT）模型：来源、人为创造、底层原理

## 1. 模型怎么来的？是不是人为创造？
**是人为推导出来的统计模型，分两段历史**
1. **1929 Zermelo（策梅洛）**：最早为国际象棋选手排名提出，用两两胜负估计选手实力，给出迭代求解方案，只是没标准化命名arXiv。
2. **1952 Bradley & Terry**：独立完整系统化，发表成对比较秩分析论文，给出完整极大似然估计框架，正式定名 Bradley-Terry 模型，成为统计学标准工具。

本质：**人为基于 “成对比较 + 单维潜实力” 假设构建的概率模型**，不是物理定律，是对人类偏好、竞技胜负的数学抽象，和 Elo 评分、Logit 奖励模型同源。
## 2. 核心数学原理

### 两层底层逻辑

1. **潜效用假设**：每个对象有唯一一维真实分数，偏好只由分数差决定；
2. **逻辑噪声假设**：人类对比时存在逻辑分布随机噪声，差值经过 sigmoid 映射成 0~1 偏好概率；
  等价理解：两个逻辑分布变量相减，胜出概率恰好是 BT 形式。

### 关键性质

- 尺度不变：所有加常数，偏好概率不变；
- 传递性：若，则模型天然推出；
- 极大似然目标：给定标注，最大化 ，就是奖励模型标准损失。

## 1. 核心架构对比

表格
| 维度 | PPO-RLHF | DPO |
| --- | --- | --- |
| 依赖模型 | 4 个：Policy、Ref、Reward Model、Critic 价值网络 | 2 个：Policy、Ref（SFT 模型） |

## 2. PPO 优缺点

### 优点

1. 在线采样，能探索 SFT 没覆盖的优质回答，复杂推理、多步骤任务上限更高；
2. 奖励函数可灵活定制（安全、工具调用、多目标加权），可控性强；
3. 理论强化学习完备，可扩展价值函数、多智能体、动态环境。

### 缺点

1. 工程极重：4 模型并行、GPU 开销大、训练链路长；
2. 训练不稳定：容易出现奖励劫持、模式崩溃、KL 剧烈波动；
3. 多轮迭代采样，样本利用率低，训练耗时久；
4. 误差累积：RM 本身有拟合误差，再传入 PPO 二次放大偏差。

## 3. DPO 优缺点

### 优点

1. 极简链路，单阶段微调，实现成本极低，单卡可跑；
2. 训练极度稳定，损失平滑，无崩溃、无剧烈 KL 震荡；
3. 计算量小，样本利用率高，开源模型主流对齐方案；
4. 无奖励模型误差累积，梯度直接作用于生成策略；
5. 基于 BT 闭式解，理论简洁，调参少（仅 β 温度系数）。

### 缺点

1. 纯离线学习，**无法自主探索新回答**，高度依赖现有偏好对覆盖范围；
2. 偏好数据质量敏感：标注噪声、循环偏好会直接损害效果；
3. 多复杂目标平衡弱于 PPO，难以精细加权多种奖励维度；
4. 长链式推理、需要主动试错的任务（数学证明、工具调用）上限略低于 PPO。

## 4. 什么时候选谁？

- **选 DPO**：通用对话、摘要、写作、小规模微调、开源复现、资源有限、追求稳定低成本；
- **选 PPO-RLHF**：高端商用基座、复杂推理、工具 Agent、多目标精细对齐、需要模型自主探索新行为。

# 五、整体脉络串联

1. Bradley-Terry：1929-1952 人为构建的成对比较概率模型，用潜效用差建模人类二元偏好；
2. RLHF-PPO：先用 BT 损失训练奖励模型，再用强化学习迭代最大化奖励；
3. DPO 数学突破：利用带 KL 约束 RL 最优策略闭式解，把奖励替换为策略对数比，直接代入 BT 似然；
4. 替代逻辑：二者优化目标数学等价，DPO 砍掉中间 RM 与在线强化循环，用监督式梯度完成对齐，因此能低成本替代 PPO；
5. 取舍：DPO 胜在稳定、简单、低成本；PPO 胜在探索能力、多目标精细控制。

# 二、传统方案：PPO-RLHF 完整流程（三步，本例落地）

## 步骤 1：SFT 得到参考模型 π_ref
先用优质对话数据微调基座，得到基础能正常对话的模型 `π_ref`，也就是我们的初始策略。
## 步骤 2：用 BT 损失训练独立奖励模型 RM
投喂大量成对标注，训练一个单独网络，专门给回答打分：
输入`"写一段加班安慰文案"+好文案` → 输出高分
输入`"写一段加班安慰文案"+差文案` → 输出低分
## 步骤 3：PPO 在线强化学习（核心循环）

1. **采样**：拿当前策略  对 prompt `x` 生成一堆回答，比如同时产出：
  A：温柔安慰（高分）
  B：冷漠敷衍（低分）
  C：鸡汤说教（中等分）
2. **打分**：把 A/B/C 全部丢进刚才训好的 RM，得到各自奖励 ；
3. **计算 PPO 目标**

  - 第一项：鼓励生成 RM 打分高的安慰文案；
  - 第二项 KL 惩罚：不让模型偏离原始 SFT 模型太远，防止胡说八道；
4. **更新策略**：用策略梯度更新模型，重复采样 - 打分 - 更新几十轮。

### 本例 PPO 的问题（痛点）

1. 多一套 RM 网络，显存翻倍，训练链路长；
2. RM 本身有误差：如果标注噪声大，打分不准，PPO 会学偏；
3. 在线采样要不断生成新句子，计算量大、训练不稳定；
4. 容易奖励劫持：模型钻漏洞，写出看似高分但不符合人类真实偏好的句子。

# 四、同一案例对比：PPO vs DPO 优劣直观感受

PPO 会在线生成大量全新句子，比如：
“别熬太晚，下班买点奶茶犒劳自己，明天再忙也没关系”
这类标注里不存在的优质回答，模型能自己探索出来，文案多样性、创造力更强。
适合需要发散、多步骤推理、工具调用的复杂任务。
### PPO 劣势

- 训练成本高，要同时跑策略、参考、奖励、价值 4 个模型；
- 若 RM 打分跑偏，模型会学到畸形文案；
- 训练波动大，一不小心模型全部输出空洞鸡汤。

### DPO 优势体现

- 只用标注里现成的好坏配对，一次微调完成，单卡就能跑；
- 训练曲线平滑，不会突然失控；
- 工程简单，不用维护强化学习采样循环。

### DPO 劣势
完全依赖标注数据，**不会主动创造标注外的新颖优质文案**。
如果标注里缺少生活化、多元化安慰话术，模型生成会很死板、同质化。
# 五、整条逻辑链总结（结合案例）

1. Bradley-Terry：数学工具，描述人类对比两条文案时的选择概率，是所有偏好对齐的底层假设；
2. PPO-RLHF：先用 BT 损失训独立奖励模型打分，再在线强化学习最大化分数；
3. DPO：利用 RL 闭式解消去奖励模型，直接把策略概率比当成隐性奖励，代入同一个 BT 损失优化；
4. 替代关系：两者优化目标数学等价，DPO 砍掉复杂在线强化流程，用简单监督微调实现对齐；
5. 选型：文案写作、对话微调、小资源场景选 DPO；复杂推理、需要自主创新生成内容选 PPO。

### 痛点 1：梯度会消失，训练后期很难收敛
对损失  求梯度：

当模型已经预测很准，，梯度趋近于 0，几乎不更新；
而  的梯度：

接近 1 时梯度稳定存在，模型会持续微调，打分越来越精准。
### 痛点 2：分布假设不匹配，数学上不严谨
我们的数据生成假设是 Bradley-Terry 概率分布，
统计学上，拟合该分布的标准最优目标是**最大化似然**，对应损失。
 没有任何概率分布推导支撑，只是人为构造的距离，用它训练 RM 会出现两个问题：
1. 学到的打分不满足 BT 概率逻辑，后续 PPO/DPO 对齐全部失真；
2. 梯度不平衡，模型容易卡在局部最优，打分区分度很差。

### 极简文字浓缩（方便快速记忆）

1. **噪声**：数据里没法预测的随机偏差；房价上下浮动是连续噪声，人偶尔选错文案是分类噪声。
2. **正态分布噪声**：连续数值误差大多集中在小范围，极少极端偏差，对应回归任务。
3. **极大似然**：调模型参数，让你手上真实标注数据最容易出现。
4. 统一底层：

  - 连续数值 + 正态噪声 → 极大似然推导出 MSE；
  - 二元偏好 + BT 逻辑分布噪声 → 极大似然推导出 -logP 损失；
5. MSE 只是特殊情况，NLL 是概率任务通用形式，二者不能随便互换，因为底层随机规律假设完全不同。
