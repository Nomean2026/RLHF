DPO 的核心洞察：既然我们最终要的是让模型知道“好的回答比坏的回答更受偏爱”，那为什么不直接把“偏好”这个信号灌进策略模型里，非要绕道训练一个奖励模型再强化学习呢？

🍜 通俗比喻：PPO 像是请了一位美食评论家（Reward Model）给菜打分，厨师（Actor）再根据分数和陪练（Critic）的实时反馈调整菜谱。
DPO 则像直接告诉厨师：“顾客觉得这道菜比那道菜好吃，你就照着这个方向改进，别管具体打多少分。”

💡 DPO 核心思想：把偏好概率变成损失函数

假设我们有一批人类偏好数据，每个样本包含一个提示 $x$，以及两个回答：

  • 获胜回答 $y_w$ （人类更喜欢）
  • 失败回答 $y_l$ （人类不喜欢）

DPO 直接从数学上推导出一个损失函数，让策略模型 $\pi_\theta$ 能直接利用这些偏好对进行优化，而不需要显式训练奖励模型。

📐 第一步：Bradley-Terry 偏好模型

在传统的 RLHF 中，人类偏好被建模为：

$$ P(y_w \succ y_l \mid x) = \frac{\exp\big(r(x, y_w)\big)}{\exp\big(r(x, y_w)\big) + \exp\big(r(x, y_l)\big)} $$

这里 $r(x,y)$ 是一个隐藏的奖励函数（也就是奖励模型）。
这个公式的意思是：回答 $y_w$ 打败 $y_l$ 的概率，取决于两者奖励值的指数之比。

🔁 第二步：建立奖励与策略的桥梁

在 KL 约束的强化学习目标下，最优策略 $\pi^*$ 和奖励函数 $r$ 存在一个精确的映射关系：

$$ r(x, y) = \beta \log \frac{\pi^*(y \mid x)}{\pi_{\text{ref}}(y \mid x)} + \beta \log Z(x) $$

其中 $Z(x)$ 是配分函数（只依赖于 $x$ 的归一化常数），$\beta$ 是控制 KL 惩罚强度的超参数。
这个公式太美妙了：它说“奖励”可以完全由策略模型与参考模型的概率比来表示！

🎯 第三步：代入偏好模型，消掉奖励函数

把上面的 $r$ 表达式代入 Bradley-Terry 模型，奇迹发生了——$Z(x)$ 被抵消掉：

$$ P(y_w \succ y_l \mid x) = \frac{1}{1 + \exp\left( \beta \log \frac{\pi^*(y_l \mid x)}{\pi_{\text{ref}}(y_l \mid x)} - \beta \log \frac{\pi^*(y_w \mid x)}{\pi_{\text{ref}}(y_w \mid x)} \right)} $$

现在，人类偏好概率只依赖于策略模型 $\pi^*$ 和参考模型 $\pi_{\text{ref}}$，不再需要奖励模型！

📉 第四步：最大似然估计 —— DPO 损失函数

我们希望策略模型 $\pi_\theta$ 能最大化人类偏好数据中获胜回答的似然。对上面的概率取负对数，就得到 DPO 损失：

$$ \mathcal{L}_{\text{DPO}}(\pi_\theta; \pi_{\text{ref}}) = - \mathbb{E}_{(x, y_w, y_l) \sim \mathcal{D}} \left[ \log \sigma \left( \beta \log \frac{\pi_\theta(y_w \mid x)}{\pi_{\text{ref}}(y_w \mid x)} - \beta \log \frac{\pi_\theta(y_l \mid x)}{\pi_{\text{ref}}(y_l \mid x)} \right) \right] $$

其中 $\sigma(\cdot)$ 是 sigmoid 函数。这个损失函数的直观含义是：

  • 让获胜回答的相对概率比 $\frac{\pi_\theta(y_w)}{\pi_{\text{ref}}(y_w)}$ 尽可能大
  • 让失败回答的相对概率比 $\frac{\pi_\theta(y_l)}{\pi_{\text{ref}}(y_l)}$ 尽可能小
  • 两者之间的差距越大，损失越小；模型就越能“分辨”好坏回答。

🍜 实例：夸夸机器人再就业（DPO 版）

还是那个 prompt：“我刚剪了个超丑的发型，求安慰。”

我们收集了一条人类偏好数据：

角色回答内容人类偏好
🥇 获胜 $y_w$ 别担心，新发型很衬你！ ✅ 更喜欢
🥈 失败 $y_l$ 哈哈哈哈哈哈丑哭了 ❌ 不喜欢

DPO 训练时，我们做如下计算：

  1. 用当前策略 $\pi_\theta$ 和参考模型 $\pi_{\text{ref}}$ 分别计算两个回答的对数概率。
  2. 计算相对比率：$r_w = \beta \log \frac{\pi_\theta(y_w)}{\pi_{\text{ref}}(y_w)}$，$r_l = \beta \log \frac{\pi_\theta(y_l)}{\pi_{\text{ref}}(y_l)}$。
  3. 损失 = $-\log \sigma(r_w - r_l)$。

如果模型已经能很好区分好坏（$r_w \gg r_l$），sigmoid 接近 1，损失接近 0；
如果模型还糊涂（$r_w \approx r_l$ 甚至 $r_w < r_l$），损失就会很大，反向传播会拉升 $y_w$ 的概率，压低 $y_l$ 的概率。

🧘 对比 PPO：PPO 需要先训练奖励模型给“别担心…”打 8.0 分，给“哈哈哈…”打 1.2 分，再通过 Critic 拆解成 token 级优势。
DPO 则直接说：“反正人类更喜欢第一句，你看着办。”——简单粗暴，效果却出奇地好。

⚙️ DPO 训练流程（极简版）

  1. 准备数据：收集人类偏好对 $(x, y_w, y_l)$，例如同一个 prompt 下的两个回答，标注员选出更好的那个。
  2. 初始化模型：策略模型 $\pi_\theta$ 从 SFT 模型热启动；参考模型 $\pi_{\text{ref}}$ 是 SFT 模型的冻结副本。
  3. 计算对数概率：对于一个 batch 的偏好对，分别用 $\pi_\theta$ 和 $\pi_{\text{ref}}$ 计算 $\log \pi(y_w|x)$ 和 $\log \pi(y_l|x)$。
  4. 构造 DPO 损失： $$ \mathcal{L} = -\frac{1}{N}\sum_{i=1}^N \log \sigma\left( \beta \log \frac{\pi_\theta(y_w^{(i)})}{\pi_{\text{ref}}(y_w^{(i)})} - \beta \log \frac{\pi_\theta(y_l^{(i)})}{\pi_{\text{ref}}(y_l^{(i)})} \right) $$
  5. 反向传播更新 $\pi_\theta$：优化器只更新策略模型参数，参考模型纹丝不动。
  6. 重复迭代：直到模型在偏好数据上表现稳定，生成的回答更符合人类喜好。

注意：整个过程没有奖励模型，没有 Critic，没有强化学习的“采样-评估-更新”循环。DPO 就是一个监督式的排序损失，训练速度更快，超参数更少。

💻 伪代码：DPO 训练一步

# 假设已有：policy_model (π_θ), ref_model (冻结的 SFT)
# 数据 batch: prompts, winning_responses, losing_responses

# 1. 计算对数概率
logp_w_policy = policy_model.log_prob(prompts, winning_responses)
logp_l_policy = policy_model.log_prob(prompts, losing_responses)
logp_w_ref    = ref_model.log_prob(prompts, winning_responses)
logp_l_ref    = ref_model.log_prob(prompts, losing_responses)

# 2. 计算相对比率
ratio_w = logp_w_policy - logp_w_ref
ratio_l = logp_l_policy - logp_l_ref

# 3. DPO 损失
loss = -F.logsigmoid(beta * (ratio_w - ratio_l)).mean()

# 4. 反向传播
optimizer.zero_grad()
loss.backward()
optimizer.step()

看，连 Critic 的影子都没有！代码量比 PPO 少了一个量级。

DPO 的出现让对齐训练从“重型武器”变成了“轻量飞镖”——快、准、稳。当然，如果偏好数据质量不高，DPO 也可能学偏，但这就是另一个故事了。

🧘 DPO 终极心法口诀

  1. SFT 模型 → 冻一份当 Reference（师爷）
  2. 收集 人类偏好对 $(y_w, y_l)$，不需要打分，只需要“更喜欢哪个”
  3. Policy（少侠） 从 SFT 热启动，Reference 冻住
  4. 计算两个回答的 相对概率比：$\beta \log \frac{\pi_\theta}{\pi_{\text{ref}}}$
  5. 损失函数：$-\log \sigma(\text{获胜比率} - \text{失败比率})$
  6. 反向传播，只更新 Policy
  7. 循环迭代，直到少侠本能地选择人类喜欢的回答 🎉

DPO = 偏好数据 + 对数概率减法 + Sigmoid → 少侠悟道！
