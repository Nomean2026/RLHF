🏆 GRPO · 组内比武

不要陪练，不设评委——让少侠们自己打一架，谁赢学谁

📖 缘起：PPO 的“贵族病”与 DPO 的“洁癖”

在 RLHF 的演进之路上，我们遇到了两位前辈：

  • PPO（四大天王版）：需要训练一个与策略模型同等规模的 Critic（价值模型）。这就像给每个运动员配一个专属教练——成本翻倍，内存爆炸，而且教练水平不够还会帮倒忙（价值估计偏差）。
  • DPO（直接偏好版）：虽然抛弃了 Critic 和 Reward，但它依赖静态的偏好数据集，无法在训练中在线探索新策略，容易“背题库”，分布外泛化能力受限。

于是，DeepSeek 团队在训练数学推理模型时拍案而起：“能不能只要一个 Actor，自己生成一堆答案，然后内部比个高低，直接学？”
——GRPO（Group Relative Policy Optimization，组内相对策略优化）应运而生。

🍜 灵魂比喻：PPO 是给每个学生（Actor）配一个家教（Critic）；DPO 是让学生背标准答案；GRPO 是让一群学生做同一道题，然后互相对比：“你这解法妙！”“你这思路歪了！”——同伴互评，共同进步。

💡 核心思想：组内相对优势——自己当自己的裁判

GRPO 的关键洞察是：我们并不需要知道一个回答的绝对价值（Critic 做的事），只需要知道它相对于同组其他回答有多好。

对于每个提示 $x$，GRPO 让旧策略 $\pi_{\theta_{\text{old}}}$ 独立采样 $G$ 个回答（通常 $G=4$ 或 $8$），形成一个“比武小组”。然后用奖励模型（是的，GRPO 仍需要奖励模型，但它只需要输出标量分数，不需要和 Actor 一样大！）给这 $G$ 个回答打分：$r_1, r_2, \dots, r_G$。

接下来就是 GRPO 的魔法——组内标准化：

$$ \hat{A}_i = \frac{r_i - \text{mean}(r)}{\text{std}(r)} $$

这里 $\hat{A}_i$ 就是第 $i$ 个回答的优势。
优势 > 0 → 这个回答比组内平均水平好，值得鼓励；
优势 < 0 → 这个回答拖了小组后腿，需要打压。

因为优势完全来自组内比较，GRPO 天然消除了奖励模型的绝对分值漂移——哪怕奖励模型给所有回答都打高分，只要组内相对顺序对，训练信号就是准确的。

🔮 优势不来自 Critic，来自“同行衬托”：这就像考试分数出来，你不需要知道全省平均分，只要知道你在班里排第几，就知道该不该奖励自己一顿火锅。

📐 GRPO 的数学之美：从 PPO 到组内优势

1. 回顾 PPO 的目标

PPO 的裁剪损失（简化）为：

$$ \mathcal{L}^{\text{PPO}}(\theta) = \mathbb{E}_t \left[ \min\left( \rho_t A_t, \text{clip}(\rho_t, 1-\epsilon, 1+\epsilon) A_t \right) \right] $$

其中 $\rho_t = \frac{\pi_\theta(a_t|s_t)}{\pi_{\theta_{\text{old}}}(a_t|s_t)}$ 是概率比，$A_t$ 是 Critic 估计的优势。

2. GRPO 的改造——用组内标准化代替 Critic

GRPO 保留了 PPO 的裁剪机制，但把优势 $A_t$ 替换为基于整句回答的组内标准化优势。对于第 $i$ 个回答（包含多个 token），GRPO 定义其优势为：

$$ \hat{A}_{i} = \frac{r_i - \text{mean}(\{r_1, ..., r_G\})}{\text{std}(\{r_1, ..., r_G\})} $$

然后，这个回答内的每一个 token 都共享同一个优势 $\hat{A}_i$（即句子级优势，而非 token 级）。策略梯度目标变为：

$$ \mathcal{L}^{\text{GRPO}}(\theta) = \mathbb{E}_{x, \{y_i\}_{i=1}^G \sim \pi_{\theta_{\text{old}}}} \left[ \frac{1}{\sum_i |y_i|} \sum_{i=1}^G \sum_{t=1}^{|y_i|} \min\left( \rho_{i,t} \hat{A}_i, \text{clip}(\rho_{i,t}, 1-\epsilon, 1+\epsilon) \hat{A}_i \right) \right] $$

🍜 实例：夸夸机器人 GRPO 训练现场

Prompt：“我刚剪了个超丑的发型，求安慰。”

旧策略 $\pi_{\theta_{\text{old}}}$ 一口气生成了 $G=4$ 个回答：

回答编号回答内容奖励分 $r$
🥇 y₁ 别担心，新发型很衬你！ 8.2
🥈 y₂ 没事的，我也刚剪残，一起丑 6.8
🥉 y₃ 哈哈哈哈哈哈丑哭了 1.5
4️⃣ y₄ 换个发型师吧，真的 4.0

组内标准化：均值 $\mu = (8.2+6.8+1.5+4.0)/4 = 5.125$，标准差 $\sigma \approx 2.56$。

回答原始奖励 $r$优势 $\hat{A} = (r-\mu)/\sigma$信号含义
y₁8.2+1.20👍 明显优秀，鼓励
y₂6.8+0.65🙂 还不错，稍微鼓励
y₃1.5-1.42👎 垫底，狠狠打压
y₄4.0-0.44😐 略低于平均，轻微打压

然后，GRPO 用这些优势值去更新策略：y₁ 中每个 token 的概率比乘以 +1.20，被裁剪后提上去；y₃ 中每个 token 的概率比乘以 -1.42，被压下来。没有 Critic 的任何参与，组内对比自动给出了清晰的梯度方向。

⚙️ GRPO 训练流程（单步循环）

  1. 采样小组：从一个 batch 的 prompts 中，对每个 prompt $x$，用当前策略 $\pi_{\theta_{\text{old}}}$ 独立采样 $G$ 个回答 $\{y_1, ..., y_G\}$。
  2. 奖励打分：用预先训练好的奖励模型（Reward Model）为每个 $y_i$ 计算标量奖励 $r_i$。
  3. 组内标准化：对每个 prompt 的 $G$ 个奖励，计算均值 $\mu$ 和标准差 $\sigma$，得到优势 $\hat{A}_i = (r_i - \mu)/\sigma$。
  4. （可选）加入 KL 惩罚：可以修改 $r_i$ 减去 KL 散度，或者在损失函数中单独加 KL 正则项。
  5. 计算概率比：用当前策略 $\pi_\theta$ 和旧策略 $\pi_{\theta_{\text{old}}}$ 分别计算所有回答中每个 token 的对数概率，得到 $\rho_{i,t}$。
  6. 构造 GRPO 损失： $$ \mathcal{L} = -\frac{1}{\sum_i |y_i|} \sum_{i=1}^G \sum_{t=1}^{|y_i|} \min\left( \rho_{i,t} \hat{A}_i, \text{clip}(\rho_{i,t}, 1-\epsilon, 1+\epsilon) \hat{A}_i \right) + \gamma \cdot \text{KL} $$
  7. 反向传播更新 $\pi_\theta$：优化器更新策略参数。旧策略 $\pi_{\theta_{\text{old}}}$ 定期同步为当前策略。
  8. 重复：直到策略在组内对比中表现稳定。

💻 伪代码：GRPO 一步更新

# 输入：policy_model (π_θ), ref_model, reward_model, prompts batch # 超参数：G=4, ε=0.2, β=0.04 (KL系数) # 1. 采样小组 old_policy = policy_model.copy() # 冻结旧策略 all_responses = [] for prompt in prompts: responses = old_policy.sample(prompt, num_samples=G) all_responses.append(responses) # 2. 计算奖励 rewards = reward_model(prompts, all_responses) # shape: [batch_size, G] # 3. 组内标准化得优势 mean_r = rewards.mean(dim=1, keepdim=True) std_r = rewards.std(dim=1, keepdim=True) + 1e-8 advantages = (rewards - mean_r) / std_r # 4. 计算概率比和 KL logp_old = old_policy.log_prob(prompts, all_responses) logp_ref = ref_model.log_prob(prompts, all_responses) logp_curr = policy_model.log_prob(prompts, all_responses) ratio = torch.exp(logp_curr - logp_old) kl = logp_ref - logp_curr # 近似 KL # 5. GRPO 损失 surr1 = ratio * advantages.unsqueeze(-1) # 广播到 token 维度 surr2 = torch.clamp(ratio, 1-ε, 1+ε) * advantages.unsqueeze(-1) policy_loss = -torch.min(surr1, surr2).mean() kl_loss = β * kl.mean() loss = policy_loss + kl_loss # 6. 更新 optimizer.zero_grad() loss.backward() optimizer.step()

注意：GRPO 完全没有 Critic 网络，优势直接通过组内奖励标准化得到，代码量比 PPO 精简了整整一个 Critic 的部分。

GRPO 站在 PPO 和 DPO 的肩膀上：保留了 PPO 的在线探索能力，又像 DPO 一样不需要 Critic，同时用组内标准化巧妙地解决了绝对奖励的尺度问题。

🧘 GRPO 终极心法口诀

  1. 旧策略 对一个 prompt 生成 $G$ 个回答（组内比武）
  2. 奖励模型 给每个回答打一个总分 $r_i$
  3. 组内标准化： $\hat{A}_i = (r_i - \mu)/\sigma$，同行衬托出优势
  4. 优势 > 0 的整句回答，所有 token 一起提概率；优势 < 0 的，一起压概率
  5. 用 PPO 裁剪防止步子太大，加 KL 惩罚防止忘了师爷
  6. 没有 Critic，没有绝对分值依赖，只有组内相对排名
  7. 循环迭代，少侠们在互相比拼中集体进化 🚀

GRPO = 小组采样 + 组内标准化 + PPO裁剪 → 不需要陪练的强化学习！
