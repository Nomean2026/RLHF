📖 SFT · 有监督微调

预训练才子的"私教课"——从"满嘴跑火车"到"好好说话"

🎓 SFT 是什么？

预训练模型读了海量文本，知道语言的统计规律，但不会"按指令办事"。你问它"帮我写首诗"，它可能继续补全网页文本。

SFT（Supervised Fine-Tuning）就是用人工标注的"指令-回答"对，教模型学会对话格式和指令遵循。

🍜 通俗比喻：预训练模型是一个读了万卷书的才子，才华横溢但说话不着调。SFT 就是请了一位私教，拿标准答案一句句教他："人家问你好，你得说'你好！有什么可以帮您？'而不是继续背课文。"

SFT 是 RLHF 家族所有后续方法的起点——PPO 的 Actor 从 SFT 热启动，DPO 的 Policy 从 SFT 开始，GRPO 同理。没有 SFT，后面的强化学习就是"教一个不会说话的模型写文章"。

💡 核心思想：只教该教的

一段对话数据长这样：

<|user|>帮我翻译这句话<|assistant|>好的，请把原文发给我。

关键设计：只对 assistant 的回复部分计算损失，user 的输入被标记为忽略（-100）。这样模型只学"怎么回答"，不学"怎么提问"。

📐 损失函数

$$ \mathcal{L}_{\text{SFT}} = -\sum_{t \in \text{assistant}} \log P_\theta(y_t \mid y_{<t}, x) $$

就是标准的交叉熵损失，只不过 mask 掉了 user 部分的 token。

🏗️ 训练流程

  1. 准备数据：收集高质量的指令-回答对，格式化为特殊 token 分隔的对话序列。
  2. 构建标签：将 user 部分标记为 -100（忽略），只保留 assistant 回复作为训练目标。
  3. 加载预训练模型：从预训练 checkpoint 热启动，模型已经"会说话"，只是"不听话"。
  4. 微调训练：用较小的学习率（如 $1\text{e-}5$）训练几个 epoch，让模型学会按指令回答。
  5. 保存模型：产出 SFT 模型（后续作为 Actor/Policy 初始）和冻结副本（作为 Reference）。

💻 核心代码

# 只对 assistant 部分计算损失
logits = model(x)  # [B, S, vocab_size]
loss = F.cross_entropy(
    logits.view(-1, vocab_size),
    labels.view(-1),
    ignore_index=-100  # user 部分不参与梯度
)

标签中 -100 的位置会被 PyTorch 自动忽略，梯度只从 assistant 的回复中产生。

🔗 SFT 在 RLHF 家族中的位置

后续方法SFT 的角色
PPOSFT 模型作为 Actor 热启动；冻一份当 Reference
DPOSFT 模型作为 Policy 初始；冻一份当 Reference
GRPOSFT 模型作为 Policy 初始；冻一份当 Reference
⚠️ SFT 的局限：SFT 只能"模仿"人类标注的答案，无法超越标注质量。如果标注员的回答只有 70 分，SFT 模型最多也到 70 分。这就是为什么我们需要 PPO、DPO、GRPO 来进一步对齐——它们的目标是让模型探索出比标注更好的策略。

🧘 SFT 终极口诀

  1. 预训练模型会说话但不听话
  2. 收集指令-回答对，格式化为特殊 token 序列
  3. 标签中 user 部分标 -100，只对 assistant 算损失
  4. 小学习率微调几个 epoch
  5. 产出 SFT 模型（可训练）和 Reference 模型（冻结）
  6. SFT 是所有后续对齐方法的起点 🚀

SFT = 预训练模型 + 标准答案 + 只教回答 = 好好说话！
